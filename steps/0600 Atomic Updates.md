目前实现的数据库采用日志加上内存数组的结构：数据全部存储在内存数据中，在对数据库进行修改时会使用日志记录，且在启动数据库时会读取日志并恢复数据。当前实现的数据库，其存储数据受限于内存大小，因此必须设计一个基于磁盘的数据结构，将一定量的数据保存到磁盘。

无论使用何种数据结构，一旦数据存储到磁盘上，就必须考虑原子性和持久性。日志通过校验和实现原子性：追加数据不会破环旧状态，校验和可以检测到不完整的新状态。

若考虑将当前内存数组直接存储到磁盘上，就要考虑数组的插入和删除操作会移动元素并破坏原有数组。因此此情况下，不能像日志记录的方式保持原子性，因为将内存数组保存到磁盘时突然发生断电，可能会导致整个存储内容不可用。为避免破坏旧状态，可以复制数据，应用更新时直接一步将新数组替换旧数组。替换步骤可以使用 Linux 提供的 API 来实现原子操作，该方式称为写时复制。

还有另一种不依赖文件系统或硬件原子性的方法：

1. 存储两份数据副本，轮流更新它们。
2. 每个副本都有一个单调递增的版本号，使用版本号较大的副本。
3. 每个副本都带有校验和，因此在恢复后可以忽略可能损坏的副本。
